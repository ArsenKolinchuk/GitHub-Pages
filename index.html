Що таке система контролю версій?

Система контролю версій (VCS) дозволяє відслідковувати зміни у коді або файлах проекту. Git — одна з найпопулярніших розподілених VCS, яка використовується для відслідковування змін в коді, ведення історії змін і роботи з гілками.

Гілки та злиття

Легке перемикання між гілками та злиття змін.

Базові команди Git

  1. git init - Ініціалізація нового репозиторію Git в поточній папці.
  2. git clone [url] - Клонування існуючого репозиторію з віддаленого сервера.
  3. git status - Перевірка стану файлів у робочій директорії.
  4. git add [файл] - Додавання файлу до індексації (для наступного коміту).
  5. git commit -m "[повідомлення]" - Збереження змін у репозиторії з коментарем.
  6. git push - Відправка локальних змін на віддалений репозиторій.
  7. git pull - Оновлення локальної версії з віддаленого репозиторію.
  8. git branch - Перегляд гілок у репозиторії.
  9. git checkout [назва гілки] - Перемикання між гілками.
  10. git merge [назва гілки] - Злиття іншої гілки в поточну.
