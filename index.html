Вивчення Git

Коротка інформація про системи контролю версій

Система контролю версій (VCS) дозволяє відслідковувати зміни у коді або файлах проекту. Git — одна з найпопулярніших розподілених VCS, яка використовується для відслідковування змін в коді, ведення історії змін і роботи з гілками.

Переваги Git

  • Розподілена система: кожен учасник має повну копію історії проекту.
  • Офлайн робота: можливість роботи без підключення до мережі.
  • Гілки та злиття: легке перемикання між гілками та злиття змін.

10 базових команд Git

  1. git init - Ініціалізація нового репозиторію.
  2. git clone [url] - Клонування репозиторію.
  3. git status - Перевірка стану файлів.
  4. git add [файл] - Додавання файлу до індексації.
  5. git commit -m "[повідомлення]" - Збереження змін.
  6. git push - Відправка змін на віддалений репозиторій.
  7. git pull - Оновлення локальної версії з віддаленого репозиторію.
  8. git branch - Перегляд гілок.
  9. git checkout [назва гілки] - Перемикання між гілками.
  10. git merge [назва гілки] - Злиття іншої гілки в поточну.
